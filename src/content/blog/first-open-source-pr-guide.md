Open source can feel overwhelming, so we simplify it.

## Starter plan

- Pick a repository with beginner-friendly labels
- Reproduce the issue locally before changing code
- Keep your PR scope small and focused
- Explain what you changed and why

A clean first PR builds confidence fast.
